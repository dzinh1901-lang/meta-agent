# Operating Cadence

## Daily

- Scan project health changes.
- Identify new blockers.
- Update approval queue.
- Flag overdue approvals.
- Summarize production, billing, security, procurement, and marketing risks.

## Weekly

- Produce portfolio operating report.
- Rank projects by readiness, value, risk, and blocker severity.
- Review project milestones.
- Review procurement queue.
- Review marketing campaign readiness.
- Review security/compliance issues.

## Monthly

- Review policy matrix.
- Review approval decisions and near misses.
- Review project resource allocation.
- Update repository registry.
- Validate agent definitions and orchestration boundaries.

## Incident Cadence

For high/critical incidents:

1. Pause related agent routes.
2. Collect evidence.
3. Notify required approver roles.
4. Produce incident packet.
5. Execute only approved rollback/remediation.
6. Record post-incident review.
